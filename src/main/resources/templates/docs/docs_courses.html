Courses

This document was designed to help a client to access a management system via an HTTP API.

  • Create Course
    • REST method:
      • POST
    • Parameters:
      • JSON Object { "course_id":course_id, "name":course_name, "acronym":course_acronym, "coordinator_id":(use the pretended Teacher id ), "active":true/false (used to "remove" the course) }
    • Result:
      • The Course id.

  • Update Course
    • REST method:
      • POST
    • Parameters:
      • JSON Object { "course_id":course_id, "name":course_name, "acronym":course_acronym, "coordinator_id":(use the pretended Teacher id ), "active":true/false (used to "remove" the course) }
    • Result:
      • The Course id.

  • Activate Course
    • REST method:
      • PUT
    • Parameters:
      • course_id={course_id}
      • active={true/false ( enable/disable course )}
    • Result:
      • The Course id.

  • none
    • REST method:
      • GET
    • Parameters:
      • course_id={course_id}
    • Result:
      • JSON Object { "course_id":course_id, "name":course_name, "acronym":course_acronym, "coordinator_id":(use the pretended Teacher id ), "active":true/false (used to "remove" the course) }

  • List Courses
    • REST method:
      • GET
    • Parameters:
      • p={page number}
      • l={number of rows per page}
      • sp={search parameter}
      • sv={search value (the character * is a wildcard)}
    • Result:
      • JSON Object[] ["course_id":course_id, "name":course_name, "acronym":course_acronym, "coordinator_id":(use the pretended Teacher id ), "active":true/false (used to "remove" the course) }]
